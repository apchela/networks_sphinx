Частые проблемы и их решения

  1. Permission denied (publickey).

  • Проверить, что публичный ключ добавлен в GitHub (без лишних символов).

  • Убедиться, что ssh-agent запущен и ключ добавлен (ssh-add -l).

  • Проверить права ~/.ssh и ключа.

  • При использовании Windows PuTTY удедиться, что Pageant загружает .ppk.

  1. SSH использует не тот ключ (несколько ключей).

  • Использовать файл ~/.ssh/config, чтобы явно назначить IdentityFile для github.com.

  • Или вызвать ssh -vT git@github.com для подробного логирования — покажет, какие ключи используются.

  1. Git по-прежнему запрашивает имя/пароль.

  • Проверить URL удаленного репозитория — он должен быть SSH (git@github.com:…), а не HTTPS.

  • Проверить настройки credential helper (если Git запоминает HTTPS креденшелы).

  1. Ключ скомпрометирован.

  • Удалить соответствующий ключ из GitHub (Settings → SSH and GPG keys) и сгенерировать новый ключ; больше не использовать скомпрометированный.
